PART 2 — WHAT'S GENUINELY MISSING (vs faithful pipeline)
================================================================================

CORE PIPELINE — NOTHING MISSING. The 4 stages (M parallel agentic agents,
mean ensemble, agentic supervisor with confidence label, Platt α=√3)
are all implemented and validated end-to-end. The decision rule
(high-confidence → supervisor; else → mean) is implemented correctly.

DATA SOURCES — ONE IRREPRODUCIBLE GAP:
  • Internal news corpus (§4.1): Bridgewater's proprietary curated news.
    No public substitute. We use only Tavily. The paper found this corpus
    was used IN ADDITION to ProviderA, so they had TWO sources. We have ONE.
    Mitigation: not possible — proprietary asset.

VALIDATION UTILITY GAPS:
  • Foreknowledge-bias judge NOT INVOKED in pipeline (prompt exists in
    aia_prompts.py). For live eval this doesn't matter. For retrospective
    smoke audit on settled events, we'd want to invoke it post-hoc on each
    agent's reasoning trace.
  • Pairwise p-values, simplex regression, bootstrap CIs — all academic
    analysis utilities, not part of the agent pipeline. We don't need them
    for deploy.

PIPELINE EXTRAS WE ADD (beyond faithful paper):
  • Multi-outcome handling (paper is binary-only)
  • Kalshi historical candlestick fetcher
  • Per-event real_resolution_dates manual map (smoke test)
  • RFC 2822 parser for Tavily dates (paper doesn't document this issue)
  • Last-iter forced submission (safety against running out of agent iters)
  • SAVE_TRACES toggle (per-event JSON dump of full agent reasoning)
  • Multi-vendor ensemble mix (Opus + GPT-5 + Gemini) for diversity per
    Silicon Crowd paper. Paper uses single-model (o3) ensemble.

PART 4 — SHOULD WE PATCH ANYTHING BEFORE DEPLOY?
================================================================================

LOW-RISK PATCHES (cheap):
  1. INVOKE the foreknowledge judge in retrospective mode for the smoke
     audit (~30 min). Validates that none of our 26 sample events have
     foreknowledge leakage in agent reasoning.
  2. Add MEDIAN as alternative aggregator alongside mean (~10 min).
     Paper says equivalent; harmless option.

MEDIUM-RISK PATCHES (would deviate from paper):
  3. Add per-category α blend with Kalshi (agent_aia_pro already does this).
     Paper uses single α (or none); we'd be diverging.
  4. Add tiered confidence guardrail. Paper has no equivalent.
     agent_aia_pro already does this.

THINGS WE SHOULD NOT TRY TO REPLICATE:
  • Internal news corpus — proprietary
  • ForecastBench numbers — we score on our 26-event smoke instead

CONCLUSION:
  agent_aia.py is a FAITHFUL architectural reproduction. The "missing"
  items are either (a) proprietary (internal news corpus), or (b)
  academic analysis utilities not needed for production deploy. The
  pipeline does what the paper says it does, in the order it says.

  agent_aia_pro.py adds 3 v8-style calibration knobs on top (per-category
  α, tiered guardrail, Platt α=2.0). It's a hybrid, not a paper-faithful
  reproduction.
